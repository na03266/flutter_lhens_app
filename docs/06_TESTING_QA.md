# 06. 테스트 및 QA 가이드 (Testing & QA)

## 1. 테스트 환경
*   **Unit/Widget Test**: Flutter 내장 `test` 라이브러리 사용.
*   **Integration Test**: 실제 기기 또는 에뮬레이터 필요.

## 2. 테스트 실행 방법
소스코드 루트(`G:\lhens_app`)에서 실행:
```bash
# 전체 테스트 실행
flutter test

# 특정 파일 테스트
flutter test test/widget_test.dart
```

## 3. 핵심 사용자 시나리오 (UAT Checklist)
배포 전 반드시 확인해야 할 체크리스트입니다.

| ID | 시나리오 | 기대 결과 | 통과 여부 |
|:---:|:---|:---|:---:|
| TC-01 | **로그인** | 아이디/비밀번호 입력 후 메인 화면 진입 성공, 토큰 저장됨 | |
| TC-02 | **자동 로그인** | 앱 재실행 시 로그인 화면 없이 메인 화면 진입 | |
| TC-03 | **게시글 작성** | 글 작성 후 목록 최상단 노출, 상세 진입 가능 | |
| TC-04 | **채팅방 생성** | 사용자 초대 후 채팅방 생성 완료, 메시지 전송 및 수신(양방향) | |
| TC-05 | **알림 수신** | 앱 백그라운드 상태에서 FCM 푸시 알림 도착 및 탭 시 해당 화면 이동 | |
| TC-06 | **오프라인 대응** | 네트워크 연결 해제 시 적절한 에러 메시지(Toast/Dialog) 표시 | |

## 4. 결함 등급 기준 (Defect Priority)
*   **P1 (Critical)**: 앱 구동 불가, 로그인 불가, 데이터 유실 발생. **(즉시 픽스/배포 중단)**
*   **P2 (Major)**: 주요 기능(게시글 작성, 채팅) 동작 실패, 우회 방법 없음.
*   **P3 (Minor)**: UI 깨짐(오타, 레이아웃), 간헐적 버그, 사용성 불편.
